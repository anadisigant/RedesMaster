1. Endereçamento Classful (Classes A, B e C)
No início do IPv4, o endereçamento era fixo por classes.
Classes principais:
• Classe A → redes muito grandes
• Classe B → redes médias
• Classe C → redes pequenas
Exemplo:
Classe Faixa Máscara padrão
A 1.0.0.0 – 126.0.0.0 /8
B 128.0.0.0 – 191.255.0.0 /16
C 192.0.0.0 – 223.0.0.0 /24
Problema do Classfull
O grande problema era o desperdício de IPs:
• Empresas pequenas recebiam Classe B → milhares de IPs sobrando
• Classe C era pequena demais para muitos casos
• Não havia flexibilidade
2. Subnetting (Subnetting clássico)
Para resolver o desperdício dentro das redes, surgiu o subnetting.
Ele permitiu:
• dividir uma rede grande em redes menores
• “emprestar bits” da parte de host
Melhorias do subnetting
• melhor organização interna da rede
• redução de broadcast
--- PAGE 2 ---
• mais controle administrativo
Limitação
Mesmo com subnetting:
• todas as sub-redes tinham o mesmo tamanho
• ainda havia desperdício quando redes tinham necessidades diferentes
Sub-redes IPv4
O que é Sub-rede no IPv4?
Uma sub-rede (subnetwork) é uma divisão lógica de uma rede IP maior 
em redes menores.
No IPv4, o subnetting é utilizado para:
• organizar melhor a rede 
• reduzir tráfego de broadcast 
• melhorar desempenho 
• facilitar administração 
• aumentar segurança 
• aproveitar melhor os endereços IP

3. CIDR (Classless Inter-Domain Routing)
O CIDR eliminou o conceito de classes fixas.
Benefícios do CIDR:
• elimina classes A, B e C
• permite prefixos flexíveis (/13, /21, /27 etc.)
• melhora o roteamento na Internet
• reduz tamanho das tabelas de roteamento
Estudo de uma rede CIDR /27
Vamos analisar uma rede completa usando: 192.168.10.0/27
1. O que significa /27?
/27
• 27 bits são de rede
• 5 bits são de host
2. Máscara de sub-rede
255.255.255.224
3. Tamanho da sub-rede
256-224=32
Cada sub-rede possui 32 endereços IP
4. Quantidade de hosts válidos
2^5-2=30
30 hosts utilizáveis

Faixa de IPs válidos
192.168.10.1  até  192.168.10.30
Broadcast
192.168.10.31

255.255.255.224
Resumo 
Uma rede CIDR /27:
• possui 32 endereços no total
• permite 30 hosts utilizáveis
• usa máscara 255.255.255.224
• é ideal para redes pequenas (setores, laboratórios, IoT)
4. VLSM (Variable Length Subnet Mask)
O VLSM é uma evolução do subnetting dentro do CIDR.
Ele permite:
• sub-redes de tamanhos diferentes na mesma rede
Vantagem do VLSM
• máximo aproveitamento de IPs
• redes sob medida
• menos desperdício
O que é VLSM?
VLSM significa: Variable Length Subnet Mask

Endereçamento IPv6
O IPv6 (Internet Protocol version 6) foi criado 
para substituir o IPv4, principalmente por causa 
da falta de endereços disponíveis no IPv4.
Enquanto o IPv4 usa 32 bits, o IPv6 usa 128 bits, 
permitindo uma quantidade gigantesca de endereços.
1. Estrutura do Endereço IPv6
Um endereço IPv6 possui:
• 128 bits
• Representação hexadecimal
• 8 grupos de 16 bits
• Cada grupo separado por :
Exemplo:
2001:0db8:85a3:0000:0000:8a2e:0370:7334
Cada bloco possui 4 dígitos hexadecimais.
2. Sistema Hexadecimal no IPv6
O IPv6 utiliza hexadecimal:
Decimal Hexadecimal
10 A
11 B
12 C
13 D
14 E
15 F
--- PAGE 15 ---
Assim:

ABCD
são valores válidos.
3. Simplificação de Endereços IPv6
Os endereços IPv6 podem ser simplificados.
Regra 1 — Remover zeros à esquerda
2001:0db8:0000:0000:0000:ff00:0042:8329
fica:
2001:db8:0:0:0:ff00:42:8329
Regra 2 — Substituir sequências de zeros por ::
2001:db8:0:0:0:ff00:42:8329
fica:
2001:db8::ff00:42:8329
Importante
O :: só pode aparecer uma vez no endereço.
Errado:

Protocolos de Roteamento
Os protocolos de roteamento permitem que 
roteadores:
• descubram redes
• escolham melhores caminhos
• atualizem tabelas de roteamento 
automaticamente
Sem eles:
• seria necessário configurar rotas manualmente.
1. Objetivo dos Protocolos de Roteamento
Os protocolos ajudam os roteadores a responder:
Qual o melhor caminho até a rede de destino?
2. Principais Tipos de Protocolos
Existem dois modelos clássicos:
Tipo Ideia principal
Vetor de Distância envia distância e direção
Estado de Enlace divulga estado da rede
3. Vetor de Distância (Distance Vector)
No modelo vetor de distância:
• cada roteador conhece apenas seus vizinhos
• envia periodicamente sua tabela de rotas
--- PAGE 34 ---
• informa:
• distância
• direção
4. Ideia Básica do Vetor de Distância
Cada roteador pergunta aos vizinhos:
Quais redes você conhece?
5. Métrica
A decisão normalmente usa:
• número de saltos (hops)
Quanto menor:
• melhor a rota.
6. Funcionamento do Vetor de Distância
Exemplo:
A ----- B ----- C
• B informa a A que conhece C
• A aprende rota para C via B
7. Atualizações Periódicas
Os roteadores:
• enviam tabelas completas periodicamente
• mesmo sem mudanças
Isso gera:
--- PAGE 35 ---
• tráfego extra
• convergência mais lenta
8. Algoritmo Bellman-Ford
Os protocolos vetor de distância usam:
D(x,y)= min_v{c(x,v)+D(v,y)}
Ideia:
• escolher o menor custo através dos vizinhos.
 Exemplo: RIP
O principal protocolo vetor de distância é:
RIP
Características:
• usa hops
• máximo: 15 hops
• convergência lenta
• simples
12. Atualização RIP
